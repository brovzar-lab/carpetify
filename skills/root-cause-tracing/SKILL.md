# Root Cause Tracing

Use when errors occur deep in execution and you need to trace back to find the original trigger.

## When to Use This Skill

- Use when errors occur deep in execution and you need to trace back to find the original trigger.

## Instructions

1. Collect the full error trace and stack frames
2. Walk backwards through the call chain to identify the root trigger
3. Check for environmental factors (config, state, timing)
4. Produce a root-cause analysis with evidence chain
5. Recommend fix at the root, not the symptom
