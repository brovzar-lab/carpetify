**4. runCoherenceCheck function (Pass 2):**
- Load `revision_coherencia.md` prompt
- Build user message from: (a) all findings summaries from Pass 1 concatenated, (b) key excerpts from each reviewed document (~500 chars each)

- Parse JSON response into `CoherenceContradiction[]`
- Limit to 5 contradictions max

**5. computeReadiness function:**
Determine readiness level based on findings:
- `'lista'`: 0-3 total findings AND 0 coherence contradictions
- `'casi_lista'`: 4-6 total findings OR 1 coherence contradiction
- `'necesita_trabajo'`: 7-12 total findings OR 2-3 coherence contradictions
- `'no_lista'`: >12 total findings OR >3 coherence contradictions

**6. computeEstimatedScore function:**
Start at 100, subtract points based on findings:
- Each finding: -2 points (max deduction: findings.length * 2, capped so score >= 0)
- Each coherence contradiction: -3 points
- Clamp to 0-100 range via Math.max(0, Math.min(100, score))
