  <action>
    Create `src/hooks/useValidation.ts`.

    This hook assembles a ProjectDataSnapshot from multiple Firestore sources and runs validation rules in two tiers per D-11:

    **Tier 1 (Instant rules -- 12 rules):** Fire on every data change with 300ms debounce.
    **Tier 2 (Medium rules -- VALD-10, VALD-11):** Fire only when generatedDocs timestamps change (document generation/edit).

    **Data composition -- exact Firestore paths for financial totals:**

    1. **budgetTotalCentavos**: Read from `projects/{projectId}/meta/budget_output` document, field `totalCentavos` (integer centavos). This is written by the lineProducer pass via `storeBudgetOutputForDownstream()` and by the frontend `useBudgetEditor` auto-save. Subscribe via onSnapshot.

    2. **cashFlowTotalCentavos**: Read from `projects/{projectId}/generated/A9d` document, field `content.structured.grandTotal` (integer centavos). Written by the financeAdvisor pass. Subscribe via onSnapshot.

    3. **esquemaTotalCentavos**: Read from `projects/{projectId}/generated/E1` document, field `content.structured.total_centavos` (integer centavos). Written by the financeAdvisor pass. Subscribe via onSnapshot.

    **Other data sources:**
    - Project metadata: onSnapshot on `projects/{projectId}` (title, category, period, financial breakdown fields)
    - Team members: onSnapshot on `projects/{projectId}/team` subcollection
    - Uploaded documents: onSnapshot on `projects/{projectId}/documents` subcollection
    - Generated documents: from `useGeneratedDocs(projectId)` (already real-time)
    - ERPI settings: onSnapshot on `erpiSettings/default` document
    - Fees from contracts/budget/cash flow: extracted from generated docs content (C2b for contracts, A9b content for budget fees, A9d content for cash flow fees)

    **Tiered execution logic:**

    ```typescript
    export function useValidation(projectId: string): {
      report: ValidationReport | null
      viabilityScore: ScoreCategory[]
      improvements: ImprovementSuggestion[]
      loading: boolean
    }
    ```

    1. Assemble ProjectDataSnapshot from all data sources above.
    2. Track `generatedDocsTimestamp` -- the max `generatedAt` across all generatedDocs.
    3. When any data source changes:
       - Debounce by 300ms (per RESEARCH.md Pitfall 6)
       - Run `runInstantRules(snapshot)` to get instant results
    4. When `generatedDocsTimestamp` changes (new value different from previous):
       - Run `runMediumRules(snapshot)` to get medium results
       - Store medium results in state
    5. Merge instant + medium results into a combined ValidationReport.
    6. Also run `computeViabilityScore(snapshot)` and `generateImprovementSuggestions()`.
    7. Handle loading states from all data sources -- only run validation when all required data has loaded.

    Use `useMemo` for validation computation and `useRef` to track the previous generatedDocsTimestamp for medium tier triggering.

    **Important:** The medium rules results persist in state between instant re-runs. When instant rules re-run (data change), the combined report merges fresh instant results with the last medium results. Medium results only refresh when generatedDocsTimestamp changes.
  </action>
