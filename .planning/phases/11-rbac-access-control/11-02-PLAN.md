1. Create `functions/src/invitations/inviteToProject.ts`:
   - Export `handleInviteToProject(uid: string, data: { projectId: string; inviteeEmail: string; role: string }, callerRole: string, projectData: FirebaseFirestore.DocumentData)`
   - Validate role is one of `['line_producer', 'abogado', 'director']` -- NOT productor (only one owner per project). Throw `HttpsError('invalid-argument', 'Rol no valido.')` if invalid.
   - Normalize `inviteeEmail` to `.toLowerCase().trim()` (Pitfall 3: email case sensitivity)
   - Check if inviteeEmail matches an existing collaborator's identity. Query `users` collection `where('email', '==', normalizedEmail)` to find if that user already exists. If the user's UID is in `projectData.collaborators` or equals `projectData.ownerId`, throw `HttpsError('already-exists', 'Este usuario ya es miembro del proyecto.')`.
   - Check for existing pending invitation: query `invitations` collection `where('projectId', '==', projectId)` AND `where('inviteeEmail', '==', normalizedEmail)` AND `where('status', '==', 'pending')`. If exists, throw `HttpsError('already-exists', 'Ya existe una invitacion pendiente para este correo.')`.
   - Create invitation document in `invitations` collection with fields: `projectId`, `projectTitle` (from `projectData.metadata.titulo_proyecto || ''`), `inviteeEmail` (normalized), `role`, `status: 'pending'`, `invitedBy: uid`, `inviterName` (from `projectData.metadata` or request auth name), `createdAt: FieldValue.serverTimestamp()`, `expiresAt` (7 days from now per D-10), `respondedAt: null`.
   - Log to `security_events` collection: `{ action: 'invite_sent', userId: uid, projectId, targetEmail: normalizedEmail, role, timestamp: FieldValue.serverTimestamp() }` (per D-17).
   - Return `{ success: true, invitationId: docRef.id }`.

2. Create `functions/src/invitations/acceptInvitation.ts`:
   - Export `handleAcceptInvitation(uid: string, userEmail: string, data: { invitationId: string })`
   - Use Firestore `runTransaction` for atomicity (Pitfall 1: collaborators/memberUIDs sync):
     - Read invitation document
     - Validate: invitation exists, status is 'pending', `inviteeEmail === userEmail.toLowerCase()` (D-09 email matching safeguard -- if not matching, throw `HttpsError('permission-denied', 'Esta invitacion fue enviada a [inviteeEmail]. Inicia sesion con esa cuenta para aceptar.')`)
     - Check expiration: if `expiresAt` is in the past, update status to 'expired' and throw `HttpsError('failed-precondition', 'Esta invitacion ha expirado. Solicita una nueva al productor.')` (per D-10).
     - Update invitation: `status: 'accepted'`, `respondedAt: FieldValue.serverTimestamp()`
     - Update project document: `collaborators.${uid}: invitation.role`, `memberUIDs: FieldValue.arrayUnion(uid)`, `updatedAt: FieldValue.serverTimestamp()`
   - Log to `security_events`: `{ action: 'invitation_accepted', userId: uid, projectId: invitation.projectId, role: invitation.role, timestamp }`.
   - Return `{ success: true, projectId: invitation.projectId }`.

3. Create a `handleDeclineInvitation` in the same file or a separate small export:
   - Validate invitation belongs to caller (email match)
   - Update invitation `status: 'declined'`, `respondedAt: FieldValue.serverTimestamp()`
   - Log to security_events
   - Return `{ success: true }`

4. Create `functions/src/invitations/revokeAccess.ts`:
   - Export `handleRevokeAccess(uid: string, data: { projectId: string; targetUserId: string }, callerRole: string)`
   - Only productor can revoke (check `callerRole === 'productor'`), else throw `HttpsError('permission-denied', 'Solo el productor puede quitar acceso.')`.
   - Cannot revoke own access (targetUserId !== ownerId check)
   - Use transaction: remove `collaborators.${targetUserId}` (use `FieldValue.delete()`) and `memberUIDs: FieldValue.arrayRemove(targetUserId)` on project document (per D-15)
   - Log to security_events: `{ action: 'access_revoked', userId: uid, projectId, targetUserId, timestamp }`
   - Return `{ success: true }`

5. Register all 4 callables in `functions/src/index.ts`:
   - `inviteToProject`: onCall, region us-central1, 60s timeout, 256MiB. Calls requireAuth, requireProjectAccess (gets role), requireRole(role, ['productor']), then handleInviteToProject.
   - `acceptInvitation`: onCall, region us-central1, 60s timeout, 256MiB. Calls requireAuth, then handleAcceptInvitation with uid and `request.auth.token.email`.
   - `declineInvitation`: onCall, region us-central1, 30s timeout, 256MiB. Calls requireAuth, then handleDeclineInvitation.
   - `revokeProjectAccess`: onCall, region us-central1, 30s timeout, 256MiB. Calls requireAuth, requireProjectAccess, requireRole(role, ['productor']), then handleRevokeAccess.
