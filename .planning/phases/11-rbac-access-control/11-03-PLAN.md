<objective>
Implement the UI enforcement layer for RBAC: project access check with AccessDenied page, role-based read-only banners, hidden actions for unauthorized roles, and dashboard filtering by membership.

Purpose: AUTH-04 requires each role to have a defined permission set visible in the UI. AUTH-06 requires that unauthorized project access shows "access denied." This plan wires the permissions model from Plan 01 into every user-facing surface.

Output: useProjectAccess hook, AccessDenied page, ReadOnlyBanner component, WizardShell with access gate and role-aware rendering, dashboard with membership-filtered project list, and role-aware generation/export screens.
</objective>

2. Create `src/hooks/useProjectAccess.ts`:
   - Import `useQuery` from `@tanstack/react-query`
   - Import `doc, getDoc` from `firebase/firestore`
   - Import `db` from `@/lib/firebase`
   - Import `useAuth` from `@/contexts/AuthContext`
   - Import `ProjectRole` from `@/lib/permissions`
   - Import `useAppStore` from `@/stores/appStore`
   - Interface `ProjectAccess`: `{ hasAccess: boolean, role: ProjectRole | null, loading: boolean, ownerName: string | null }`
   - Export function `useProjectAccess(projectId: string): ProjectAccess`:
     - Get `user` from `useAuth()`
     - Use `useQuery` with key `['project-access', projectId, user?.uid]`:
       - If no user, return `{ hasAccess: false, role: null, ownerName: null }`
       - `getDoc(doc(db, 'projects', projectId))`
       - If doc does not exist, return `{ hasAccess: false, role: null, ownerName: null }`
       - If `data.ownerId === user.uid`, return `{ hasAccess: true, role: 'productor', ownerName: user.displayName }`
       - If `user.uid in data.collaborators`, return `{ hasAccess: true, role: data.collaborators[user.uid] as ProjectRole, ownerName: null }` -- ownerName will be fetched separately if needed
       - Else return `{ hasAccess: false, role: null, ownerName: null }`
     - `enabled: !!user && !!projectId`
     - `staleTime: 30_000` (30 seconds -- avoid excessive re-fetches during wizard navigation)
   - Use `useEffect` to sync the role to appStore: when `data.role` changes, call `useAppStore.getState().setCurrentProjectRole(data.role)`. On unmount, clear it to null.
   - Return `{ hasAccess: data?.hasAccess ?? false, role: data?.role ?? null, loading: isLoading, ownerName: data?.ownerName ?? null }`
