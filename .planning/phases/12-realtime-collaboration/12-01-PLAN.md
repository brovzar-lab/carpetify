3. **Create `src/hooks/useSectionLock.ts`** per D-01, D-02, D-03, D-15, research Pattern 3:
   - Export `LockHolder` interface: `{ userId: string, displayName: string, role: string, acquiredAt: number, expiresAt: number }`
   - Export `LockState` interface: `{ isLocked: boolean, isMyLock: boolean, holder: LockHolder | null }`
   - Export `useSectionLock(projectId: string, screenId: string)` hook returning `{ lockState: LockState, acquireLock: () => Promise<boolean>, releaseLock: () => Promise<void>, renewLock: () => Promise<void> }`
   - Subscribe to lock status via `onValue(ref(rtdb, lockPath(projectId, screenId)), ...)`:
     - If data is null: unlocked state
     - If data exists but `expiresAt < Date.now()`: expired, call `remove(lockRef)`, treat as unlocked
     - Otherwise: locked, check `data.userId === user?.uid` for isMyLock
   - `acquireLock`: Use `runTransaction(lockRef, (currentData) => { ... })`:
     - If null: set lock with `{ userId, displayName, role, acquiredAt: Date.now(), expiresAt: Date.now() + 120_000 }`
     - If expired (`expiresAt < Date.now()`): take over with same shape
     - If locked by another: return `undefined` to abort transaction
     - After successful acquire: call `onDisconnect(lockRef).remove()` and store ref for cancel
     - Use `.info/serverTimeOffset` to adjust timestamps for clock skew safety (per Pitfall 4): `const offset = await get(ref(rtdb, '.info/serverTimeOffset'))` then `const serverNow = Date.now() + (offset.val() ?? 0)`
     - Return `true` if committed and userId matches, `false` otherwise
   - `releaseLock`: Cancel onDisconnect, then `remove(lockRef)`
   - `renewLock`: Update `expiresAt` to `serverNow + 120_000` via `update(lockRef, { expiresAt })` -- called by idle detection heartbeat every 30s while user is active
   - Unsubscribe on cleanup

4. **Create `src/hooks/useIdleDetection.ts`** per D-03, D-07, research Pattern 4:
   - Export `IdleCallbacks` interface: `{ onIdle: () => void, onActive: () => void, onLockTimeout: () => void }`
   - Export `useIdleDetection(callbacks: IdleCallbacks, options?: { idleMs?: number, lockTimeoutMs?: number })` hook
   - Default `idleMs = 30_000` (30s per D-07), `lockTimeoutMs = 60_000` (60s per D-03)
   - Track activity via `mousemove`, `keydown`, `mousedown`, `touchstart`, `scroll` events (passive listeners)
   - Track tab visibility via `document.visibilitychange`
   - On activity: reset both idle and lock timeout timers, call `onActive` if was idle
   - On idle timer fire: call `onIdle` (dims presence)
   - On lock timeout fire: call `onLockTimeout` (releases lock)
   - On tab hidden: start idle timer immediately (don't wait for 30s of no events)
   - On tab visible: `resetTimers()` (treat as activity)
   - Cleanup all timers and listeners on unmount
