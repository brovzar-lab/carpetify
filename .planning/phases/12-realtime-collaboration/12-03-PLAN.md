**1. Modify `src/hooks/useAutoSave.ts`** per D-12, Pitfall 5:

Add two new optional parameters and a flush function:
- `lockOwned?: boolean` -- when provided as `false`, `doSave` skips the save entirely (per D-12: no writes without lock)
- `onSaveComplete?: () => void` -- callback after successful save (resets idle lock timer)
- Add `flushAndWait(): Promise<void>` return value -- immediately executes any pending debounced save and waits for completion. Used before lock release per Pitfall 5.

Implementation:
- Store the pending data in a ref (`pendingDataRef`)
- In the `save` function, also update `pendingDataRef.current = data`
- In `doSave`, check `if (lockOwned === false) return` at the top (skip save)
- After successful save in `doSave`, call `onSaveComplete?.()` and clear `pendingDataRef`
- `flushAndWait`: if `pendingDataRef.current` exists, clear the debounce timer and call `doSave(pendingDataRef.current)` directly, returning the promise
- Update return type: `{ save, status, flushAndWait }`
- Keep ALL existing behavior unchanged when `lockOwned` is not provided (backward compat)
