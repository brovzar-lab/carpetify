## Screen Specifications

### Screen 1: Activity Tab (Actividad)

**Entry point:** "Actividad" link in WizardSidebar, below Exportar and above nothing (last item after third separator).

**Layout:**
- Full content area (same width as other wizard screens, fills space right of 240px sidebar)
- Heading "Actividad" at 20px semibold, top-left
- Filter pills row below heading, 8px gap between pills
- Scrollable feed below filters, grouped by day

**Filter Pills (per D-08):**
- Two rows of pill toggle buttons
- Row 1 -- Team member: "Todos" | [each team member by displayName]. "Todos" active by default.
- Row 2 -- Event type: "Todos" | "Ediciones" | "Generacion" | "Equipo" | "Exportacion". "Todos" active by default.
- Active pill: `bg-primary text-primary-foreground` (solid fill)
- Inactive pill: `bg-transparent border border-input text-muted-foreground hover:bg-muted` (outline style)
- Pill size: `text-sm px-3 py-1 rounded-full` (pill shape, not rectangular)
- Single-select per row (clicking a pill deselects the previous one in that row)

**Day Group Headers (per D-07):**
- Text: "Hoy", "Ayer", or formatted date "20 de marzo de 2026"
- Style: `text-xs font-semibold text-muted-foreground uppercase tracking-wide`
- Spacing: 24px margin-top before each group (except first), 8px margin-bottom

**Activity Entry (per D-07):**
- Layout: horizontal row, 16px padding vertical, bottom border `border-b border-border`
- Left: User avatar (32px circle, same `PresenceAvatar` pattern -- Google photo or initials fallback, no ring color)
- Center: Stacked text
  - Line 1: `[displayName]` at 14px semibold + `[role badge]` as `<Badge variant="outline">` at 12px
  - Line 2: Action description (Spanish summary) at 14px regular, `text-foreground`
  - Line 3 (edit events only): Changed field names in `text-muted-foreground` at 12px, comma-separated
- Right: Timestamp at 12px `text-muted-foreground`, right-aligned
  - Today: relative ("hace 2 horas", "hace 15 minutos")
  - Older: absolute ("14:30")

**Event Type Icons (left of description, 16px, text-muted-foreground):**
- `edit` events: `Pencil` (lucide)
- `generate` events: `Sparkles` (lucide)
- `invite` / `accept_invite` / team events: `UserPlus` (lucide)
- `export` events: `Download` (lucide)
- `create` events: `Plus` (lucide)
- `delete` events: `Trash2` (lucide)

**Load More (per D-09):**
- Initially loads last 7 days
- "Cargar mas" button at bottom of feed, centered, `variant="outline"` size `"sm"`
- Loading state: button text changes to "Cargando..." with `Loader2` spinner icon (animated spin)
- When no more entries: button hidden

**Empty State:**
- Icon: `Activity` (lucide), 48px, `text-muted-foreground/50`
- Heading: "Sin actividad reciente" at 20px semibold
- Body: "Los cambios realizados por el equipo apareceran aqui." at 14px `text-muted-foreground`
- Centered vertically and horizontally (same pattern as `EmptyState.tsx`)

**Loading State:**
- 5 Skeleton rows: each 48px height, full width, `rounded-md`, 8px gap between skeletons

**Badge Count on Sidebar:**
- Position: Right side of "Actividad" sidebar link, same row
- Style: `<Badge>` with `bg-primary text-primary-foreground` (solid), `text-[10px] min-w-[18px] h-[18px] px-1 rounded-full flex items-center justify-center`
- Content: Number of events since `lastViewedActivity` timestamp
- Hidden when count is 0
- Count updates in real-time via `onSnapshot`
- `lastViewedActivity` written to `userProjects/{userId}/projects/{projectId}` when user navigates to Actividad tab

### Screen 2: Invitation Deep Link Page (/invitaciones/:invitationId)

**Entry point:** Email link clicked by invitee. Route: `/invitaciones/:invitationId`

**Layout:**
- Centered card on page, max-width 480px, vertically centered
- No sidebar (standalone page, not within WizardShell)
- App header visible (with sign-in state)

**State: Not Signed In**
- Card with `CardHeader` + `CardContent`
- Icon: `Mail` (lucide), 48px, `text-primary`, centered
- Heading: "Te invitaron a colaborar" at 20px semibold, centered
- Body: "Inicia sesion con tu cuenta de Google para ver la invitacion." at 14px `text-muted-foreground`, centered
- CTA: "Iniciar sesion con Google" button, `variant="default"`, full width, same as LoginPage pattern
- Below CTA: "Carpetify - Lemon Studios" at 12px `text-muted-foreground`, centered

**State: Signed In, Invitation Valid**
- Card with `CardHeader` + `CardContent` + `CardFooter`
- Heading: "Invitacion a colaborar" at 20px semibold
- Detail rows (14px, label-value pairs):
  - "Proyecto:" `[projectTitle]` (semibold value)
  - "Invitado por:" `[inviterName]`
  - "Rol:" `<Badge variant="outline">[rolLabel]</Badge>`
  - "Expira:" `[formatted date]` in `text-muted-foreground`
- Footer: Two buttons, right-aligned, 8px gap
  - "Rechazar" -- `variant="outline"` (not destructive, low visual weight)
  - "Aceptar" -- `variant="default"` (primary action)
- Loading state during accept/decline: button text changes, spinner icon, both buttons disabled

**State: Signed In, Email Mismatch (per D-09)**

- Heading: "Cuenta incorrecta" at 20px semibold, centered

**State: Invitation Expired**

- CTA: "Ir a Mis Proyectos" link button, `variant="outline"`, navigates to `/`

**State: Invitation Not Found**
- Same card layout
- Icon: `FileQuestion` (lucide), 48px, `text-muted-foreground/50`, centered
- Heading: "Invitacion no encontrada" at 20px semibold, centered
- Body: "El enlace no es valido o la invitacion fue cancelada." at 14px `text-muted-foreground`, centered
- CTA: "Ir a Mis Proyectos" link button, `variant="outline"`, navigates to `/`

**State: Invitation Already Accepted**

- Icon: `CheckCircle2` (lucide), 48px, `text-color-status-green`, centered
- Heading: "Invitacion ya aceptada" at 20px semibold, centered

## Interaction Contracts

### Activity Tab Badge

1. Badge count renders as a solid `primary` circle with white number inside, right-aligned on the sidebar "Actividad" link
2. Count = events where `createdAt > lastViewedActivity` for the current user
3. When user navigates to the Actividad tab, `lastViewedActivity` is written to `userProjects/{userId}/projects/{projectId}` with the current timestamp
4. Badge disappears when count reaches 0
5. Count updates in real-time via `onSnapshot` on the activity subcollection

### Filter Pills

1. Each filter row is single-select; clicking a pill deactivates the previous selection in that row
2. "Todos" is the default active pill in both rows
3. Clicking the already-active pill does nothing (it stays active; cannot deselect to "no filter")
4. Filters apply client-side to the already-loaded entries (no new Firestore query)
5. When both filters are active simultaneously, they combine with AND logic (e.g., "Maria" + "Ediciones" shows only Maria's edit events)

### Activity Feed Real-Time

1. New entries appear at the top of the feed without page refresh (via `onSnapshot`)
2. New entries animate in with a subtle fade (opacity 0 to 1 over 200ms via CSS transition)
3. The feed does NOT auto-scroll when new entries arrive -- the user's scroll position is preserved
4. Day group headers recalculate when the date boundary changes (midnight rollover)

### Entry Coalescing (per Pitfall 1 from RESEARCH.md)

1. Before writing a new activity entry, the client checks if the last entry from the same user on the same screen was written less than 30 seconds ago
2. If yes, the client updates the existing entry's `changedFields` array (union of fields) and `summary` instead of creating a new entry
3. If no, a new entry is created
4. This prevents activity log flood during rapid auto-save editing

### Invitation Deep Link Flow

1. User clicks email link, arrives at `/invitaciones/:invitationId`
2. If not signed in: show sign-in prompt. After sign-in, redirect back to the same invitation URL (store in `sessionStorage` before redirect)
3. If signed in but email does not match invitation's `inviteeEmail`: show email mismatch state
4. If signed in and email matches: show invitation details with Accept/Decline
5. Accept: calls `acceptInvitation` Cloud Function, shows success toast, redirects to `/project/:projectId`
6. Decline: calls `declineInvitation` Cloud Function, shows decline toast, redirects to `/`
7. If invitation expired / not found / already accepted: show the appropriate static state

### Return URL After Auth

1. When `InvitationPage` detects no auth, it stores the current URL in `sessionStorage` under key `carpetify_return_url`
2. After successful Google sign-in, `AuthContext` (or `LoginPage`) checks `sessionStorage` for `carpetify_return_url`
3. If found, navigates to that URL and removes the key
4. If not found, navigates to `/` (dashboard) as usual

## Email Template Visual Contract

The invitation email is a server-rendered HTML email (no React Email). Visual specifications:

| Property | Value |
|----------|-------|
| Max width | 600px, centered |
| Font | `-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif` (system font stack) |
| Heading | "Invitacion a colaborar" at 24px, color `#1a1a1a` |
| Body text | 16px, color `#333333` |
| CTA button | "Ver invitacion", `background: #171717` (matches oklch 0.205 primary), `color: white`, `padding: 12px 24px`, `border-radius: 6px`, `font-weight: 600` |
| Expiration note | 14px, color `#666666` |
| Footer | "Carpetify - Lemon Studios" at 12px, color `#999999`, above a 1px `#eeeeee` horizontal rule |

The email must render correctly in Gmail, Outlook, and Apple Mail. Use inline styles only, no external CSS. No images (no logo) to avoid spam filters.
