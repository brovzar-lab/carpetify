    1. Create `src/lib/export/languageCheck.ts` implementing the LANG-05 pre-export language check per D-05/D-07/D-08:

       ```typescript
       export type FindingSeverity = 'blocker' | 'flagged' | 'noted'

       export interface LanguageCheckFinding {
         type: 'anglicism' | 'currency' | 'date' | 'title'
         severity: FindingSeverity
         word: string
         context: string       // surrounding text excerpt
         docId: string
         docName: string
         replacement?: string  // suggested Spanish replacement (anglicisms only)
       }

       export interface LanguageCheckResult {
         findings: LanguageCheckFinding[]
         anglicisms: LanguageCheckFinding[]
         formatIssues: LanguageCheckFinding[]
         titleMismatches: LanguageCheckFinding[]
         passed: boolean       // true when no blockers and no flagged items
         hasBlockers: boolean  // true when title mismatches exist
       }
       ```

       Implement:
       - `ANGLICISM_BLOCKLIST`: array of `{ word: string, replacement: string, severity: 'flagged' }` with ~30 common anglicisms that should not appear in EFICINE documents. Include: 'shooting' -> 'rodaje', 'cast' -> 'elenco', 'location' -> 'locacion', 'budget' -> 'presupuesto', 'schedule' -> 'cronograma', 'deadline' -> 'fecha limite', 'feedback' -> 'retroalimentacion', 'marketing' -> 'mercadotecnia', 'target' -> 'objetivo', 'streaming' -> 'transmision', 'release' -> 'estreno', 'screening' -> 'proyeccion', 'premiere' -> 'estreno', 'workshop' -> 'taller', 'networking' -> 'vinculacion', 'performance' -> 'interpretacion', 'pitch' -> 'presentacion', 'highlight' -> 'resaltar', 'backstage' -> 'tras bambalinas', 'copyright' -> 'derechos de autor'
       - `ANGLICISM_ACCEPTED`: array of industry-accepted terms with severity:'noted': 'catering', 'DCP', 'storyboard', 'dolly', 'steadicam', 'drone', 'playback', 'claqueta', 'gaffer', 'grip', 'boom', 'travelling', 'stop motion'
       - `INVALID_CURRENCY_PATTERNS`: regex patterns that catch wrong MXN formats -- `$X.XX`, `X pesos`, `$X,XXX` without MXN suffix, numbers with decimal points that look like currency
       - `ENGLISH_DATE_PATTERNS`: regex to detect English month names (January, February, ...) in text
       - `runLanguageCheck(docs: Array<{ docId: string; docName: string; content: string }>, expectedTitle: string): LanguageCheckResult`
         - For each doc's content:
           a. Scan for anglicisms (case-insensitive word boundary match `\b${word}\b`). Create 'flagged' finding for blocklist matches, 'noted' for accepted terms.
           b. Scan for currency format violations. Look for dollar amounts not matching `\$[\d,]+\s*MXN` pattern. Flag decimals, missing MXN suffix, "pesos" instead of MXN.
           c. Scan for English date patterns (month names in English).
           d. Check title presence -- search for expectedTitle in content. If content is >100 chars and does not contain expectedTitle (case-sensitive), add a 'title' finding with severity:'blocker'.
         - Aggregate all findings. Set `passed = !hasBlockers && anglicisms.filter(a => a.severity === 'flagged').length === 0` -- but per D-07, flagged anglicisms are dismissable warnings, not blockers. So `passed` should be true only when no blockers exist. `hasBlockers` is true only when titleMismatches.length > 0.
