## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between icon and label |
| sm | 8px | Compact element spacing, form field internal padding |
| md | 16px | Default element spacing, card padding, form field gaps |

- Touch targets: minimum 36px height for all interactive elements (desktop-only app per REQUIREMENTS.md out of scope)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Form field values, table cells, paragraph text, compliance panel values |
| Label | 14px | 500 (medium) | 1.4 | Form labels, sidebar items, card metadata, section subheadings |
| Heading | 20px | 600 (semibold) | 1.2 | Screen titles ("Datos del Proyecto"), section headers, card titles |
| Display | 28px | 600 (semibold) | 1.2 | Dashboard title ("Mis Proyectos"), ERPI settings title |

**Font stack:** `Inter, system-ui, -apple-system, sans-serif`

**Rationale:** 14px body accommodates the density of a 5-screen data-entry wizard on desktop without feeling cramped. Spanish text runs longer than English, so 14px at 1.5 line height provides readability for labels like "Monto solicitado al estimulo EFICINE" without excessive line wrapping. Two weights (400 + 600) with a supporting 500 for labels keeps the type hierarchy clean per D-12 (Notion/Linear feel).

## Color

Uses shadcn/ui default theme tokens with system-preference dark mode (D-13).

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` (background) | `hsl(240 10% 3.9%)` (background) | Page background, wizard content area |
| Secondary (30%) | `hsl(240 4.8% 95.9%)` (muted) | `hsl(240 3.7% 15.9%)` (muted) | Cards, sidebar background, input backgrounds, compliance panel bg |
| Accent (10%) | `hsl(240 5.9% 10%)` (primary) | `hsl(0 0% 98%)` (primary) | See reserved list below |
| Destructive | `hsl(0 84.2% 60.2%)` | `hsl(0 62.8% 30.6%)` | Delete project button, destructive confirmation dialogs only |

**Accent reserved for (never use for general decoration):**
1. Primary CTA buttons ("+ Nuevo Proyecto" on dashboard, "Agregar aportante" when primary)
2. Active sidebar item indicator (current wizard screen)
3. Save status indicator ("Guardado" checkmark)
4. Focused form field border

**Semantic status colors (traffic light system per D-02, D-04):**

| Status | Light Value | Dark Value | Usage |
|--------|-------------|------------|-------|
| Green / success | `hsl(142 76% 36%)` | `hsl(142 70% 45%)` | Complete screen indicator, passing compliance rule, "Listo para enviar" |
| Yellow / warning | `hsl(38 92% 50%)` | `hsl(38 92% 50%)` | Partial screen indicator, warning-level compliance issue |
| Red / error | `hsl(0 84% 60%)` | `hsl(0 62% 30%)` | Error screen indicator, blocker compliance violation, missing required field |

**Source:** D-02 (traffic light status), D-12 (clean/minimal), D-13 (dark mode), D-14 (compliance panel green/red).

## Interaction Contracts

### Auto-Save (per D-03)
- Debounce: 1500ms after last keystroke
- Visual indicator: "Guardando..." during save, "Guardado" with checkmark on success, persists for 3 seconds then fades
- On error: "Error al guardar. Reintentando..." with automatic retry (3 attempts, exponential backoff)
- On unmount (screen navigation): flush pending save immediately

### Format on Blur (per D-16)
- User types raw digits: `25000000`
- On blur, field displays: `$25,000,000 MXN`
- On focus, strip formatting back to raw digits for editing
- Compliance warnings appear after blur, not during typing (per D-17)

### Traffic Light Status (per D-02)
- Green circle: all required fields on screen are valid
- Yellow circle: some fields filled, some missing or incomplete
- Red circle: validation errors present on screen
- Updates in real time as auto-save confirms data

### Delete Confirmation (per D-09)
- Dialog with destructive styling
- Confirm button uses destructive variant (red)
- Cancel button uses outline variant
- No typing confirmation required (internal tool, single user)

### Loading States
- Skeleton components for: project cards on dashboard, wizard form fields on initial load, PDF viewer area
- Skeleton duration: show for minimum 200ms to prevent flash

### Transitions
- Screen-to-screen in wizard: no animation (instant swap), content area scrolls to top
- Sidebar active state: immediate highlight change
- Dialog: shadcn default fade-in animation (150ms)
- Toast: slide in from top-right, auto-dismiss after 4 seconds
