### Side-by-Side Diff Layout

Both columns scroll synchronously. Each column is 50% width with `xl` (32px) gap between them. Minimum width per column: 300px. On viewports narrower than 700px, columns stack vertically (Version A above Version B).

Column headers:
- Left: "Version {A} — {date}" with muted-foreground text
- Right: "Version {B} — {date}" with muted-foreground text

### Structured (Table) Diff Layout

```
+---------------------------------------------------------------+
| Cuenta     | Version A          | Version B          | Diff   |
|------------|--------------------|--------------------|--------|
| 100 - Pre  | $1,200,000 MXN    | $1,350,000 MXN    | +green |
| 200 - Rod  | $3,500,000 MXN    | $3,500,000 MXN    |        |
| 300 - Post | $800,000 MXN      | $650,000 MXN      | -red   |
+---------------------------------------------------------------+
```

Uses existing Table primitive. Numeric cells right-aligned, `font-mono`. Changed cells get the appropriate diff background color. Currency formatted via `formatMXN()`.

## Interaction Contract

### Version History Toggle

1. User clicks "Historial" button (History icon) in DocumentViewer header.
2. Content area transitions to VersionHistoryPanel showing all versions.
3. "Historial" button becomes "Cerrar historial" (X icon) while panel is open.
4. Edit mode and History mode are mutually exclusive -- opening one closes the other.

### Version Comparison Flow

1. User clicks "Comparar" button at top of VersionHistoryPanel.
2. Two Select dropdowns appear (VersionSelector bar) with all versions.
3. Default selection: current (latest) version vs immediately previous version.
4. Content area shows side-by-side ProseDiffViewer or StructuredDiffViewer based on document `contentType`.
5. User can change either dropdown to compare any two versions.
6. Clicking "Cerrar" returns to normal document view.

### Revert Flow

1. User clicks "Restaurar" button next to a historical version in VersionHistoryPanel.
2. If current document has `manuallyEdited: true`, show Alert warning per D-11 first.
3. RevertConfirmDialog opens showing:
   - Target version summary (version number, date, who generated it)
   - If applicable: list of downstream documents that will become stale (per D-10)
   - Confirmation copy (see Copywriting Contract)
4. User clicks "Restaurar version" (destructive button) to confirm.
5. Cloud Function executes revert (copy-forward pattern per D-09).
6. Toast: "Version restaurada exitosamente" (sonner success).
7. DocumentViewer reloads with reverted content.
8. Activity log entry created per D-12.

### Loading States

| State | Behavior |
|-------|----------|
| Version list loading | 3x Skeleton rows (height 48px each) in VersionHistoryPanel |
| Diff computing | Skeleton block (full height of content area) with "Calculando diferencias..." label |
| Revert in progress | "Restaurar version" button shows Loader2 spinner, dialog stays open until complete |
| Version list empty (v1 only, no history) | Single entry "v1 (actual)" with no "Restaurar" button. "Comparar" button disabled with tooltip "Se necesitan al menos 2 versiones para comparar" |

### Error States

| Error | Display |
|-------|---------|
| Version fetch fails | Toast error: "No se pudo cargar el historial de versiones. Intenta de nuevo." |
| Diff computation fails | Inline Alert in diff area: "Error al calcular diferencias. Intenta seleccionar otras versiones." |
| Revert fails | Toast error: "No se pudo restaurar la version. Intenta de nuevo." Dialog closes. |
| Version not found (deleted race) | Toast error: "La version seleccionada ya no existe." |

## Accessibility Contract

| Concern | Implementation |
|---------|---------------|
| Diff color contrast | Green/red diff highlights use `-100` light backgrounds with `-900` text (WCAG AA compliant). Dark mode uses `900/30` backgrounds with `-300` text. Never rely on color alone -- deletions also use `line-through` text decoration. |
| Keyboard navigation | Version list items are focusable buttons. Tab navigates between versions. Enter triggers "Restaurar" or "Comparar". Select dropdowns use native Radix keyboard support. |
| Screen reader | Diff changes include `aria-label` on spans: "texto agregado: {word}" / "texto eliminado: {word}". Version list uses `role="list"` with `role="listitem"`. |
| Focus management | Opening RevertConfirmDialog traps focus (Radix Dialog default). Closing returns focus to the "Restaurar" button that triggered it. |
| Reduced motion | No animations in diff view. Version panel uses no transitions. |
