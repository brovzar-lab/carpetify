Fix two related bugs that break date handling for VALD-04 and document expiration:

**Bug #7 — DocumentUpload reads periodo_registro from wrong path:**
In `src/components/wizard/DocumentUpload.tsx` line 63, change:
```typescript
// WRONG:
setPeriodoRegistro(data.periodo_registro as string | undefined)

// CORRECT:
setPeriodoRegistro(data.metadata?.periodo_registro as string | undefined)
```

The project document stores `periodo_registro` under the `metadata` nested field (confirmed by `createProject` in projects.ts). Without this fix, `periodoRegistro` is always undefined and DocumentChecklist never computes expiration status.

**VALD-04 date parsing bug — engine.ts lines 377-381:**
The engine currently passes `new Date(data.metadata.periodo_registro)` as the registration close date. But `periodo_registro` contains "2026-P1" or "2026-P2", which `new Date()` cannot parse (returns Invalid Date, causing all date comparisons to return false and VALD-04 to never fail).

In `src/validation/engine.ts`, add an import at the top:
```typescript
import { PERIODOS_EFICINE } from '@/lib/constants'
```

Then create a helper function:
```typescript
/**
 * Map a periodo_registro string to its actual close date.
 * Falls back to current date if periodo not found.
 */
function getRegistrationCloseDate(periodoRegistro: string): Date {
  const periodo = PERIODOS_EFICINE[periodoRegistro as keyof typeof PERIODOS_EFICINE]
  if (periodo) {
    return new Date(periodo.close)
  }
  return new Date()
}
```

Replace the VALD-04 call (around lines 377-381) from:
```typescript
validateDateCompliance(
  data.uploadedDocs as Array<{ tipo: string; fecha_emision?: Date }>,
  data.metadata.periodo_registro
    ? new Date(data.metadata.periodo_registro)
    : new Date(),
),
```
To:
```typescript
validateDateCompliance(
  data.uploadedDocs as Array<{ tipo: string; fecha_emision?: Date }>,
  getRegistrationCloseDate(data.metadata.periodo_registro),
),
```

This ensures VALD-04 receives a valid Date object (e.g., 2026-02-13 for "2026-P1") rather than Invalid Date from `new Date("2026-P1")`.
