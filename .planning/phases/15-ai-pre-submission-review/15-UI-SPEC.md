## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, checkbox-to-label gap |
| sm | 8px | Compact element spacing, finding item padding |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, panel padding (matches existing `p-6`) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none. The existing ScoreEstimationPanel uses `p-6` (24px) and `space-y-6` (24px) which map to the `lg` token. All new components must match this spacing.

Notes:
- All numeric scores use `font-mono text-sm font-semibold tabular-nums` (matches existing ViabilityScoreCard, ArtisticScoreCard patterns).

- Section headings inside the panel use `text-sm font-semibold` (matches existing category labels), NOT `text-xl` -- reserve `text-xl` for the panel heading only.

## Interaction States

### State 1: No Review Yet (Initial)

- CTA button: "Revisar carpeta" (full-width, primary variant)
- Below button: `text-xs text-muted-foreground` disclaimer text
- If generation is incomplete: button disabled, tooltip/text below: "Genera todos los documentos antes de solicitar la revision pre-envio."

### State 2: Review In Progress (Streaming)

- CTA button: disabled, shows `RefreshCw` icon with `animate-spin` + "Evaluando carpeta..."
- Below button: progress steps displayed as `text-xs text-muted-foreground`:
  - "Cargando datos del proyecto..."
  - "Evaluando carpeta... (Evaluador 1 de 3 completado)"
  - "Evaluando carpeta... (Evaluador 2 de 3 completado)"
  - "Verificando coherencia entre documentos..."
  - "Guardando resultados..."
- Skeleton placeholders (3x `h-10 w-full`) below progress for future results area

### State 3: Review Complete (Results)

- Full results layout as described in Layout Contract above
- CTA button changes to: "Re-evaluar carpeta" (outline variant, not primary -- secondary action now)
- All findings visible with checkboxes

### State 4: Stale Review

- Yellow warning badge below readiness: `Badge` with `bg-[hsl(38_92%_50%)]/10 text-[hsl(38_92%_50%)]`
- Text: "Resultados desactualizados -- los documentos fueron regenerados despues de esta revision."
- CTA button returns to primary variant: "Re-evaluar carpeta"

### State 5: Review Error

- `Alert variant="destructive"` with `AlertCircle` icon
- Title: "La revision fallo. Intenta de nuevo."
- Retry link: "Reintentar revision" (same pattern as existing scoring error state)

### State 6: Re-evaluation Confirmation (D-13)

- Triggered when clicking "Re-evaluar carpeta" while results exist
- `Dialog` with:
  - Title: "Re-evaluar carpeta"
  - Body: "Esto generara una nueva evaluacion. Las marcas de progreso actuales se reiniciaran. Continuar?"
  - Actions: "Cancelar" (outline) | "Re-evaluar" (primary)

## Finding Item Visual Spec (ReviewFindingItem)

Each finding follows this exact layout within the 360px panel:

```
+----------------------------------------------+
| [checkbox] [persona pill] [role pill]         |
| Criterio: lenguaje cinematografico            |  <- text-xs font-medium
| Debilidad: describes camera movement but      |  <- text-xs text-muted-foreground
| not color palette or sound design.            |
| Sugerencia: add 2-3 sentences about color     |  <- text-xs text-muted-foreground
| grading approach and diegetic vs non-diegetic  |
| sound.                                         |
+----------------------------------------------+
```

- Checkbox: shadcn `Checkbox` component, 16x16px (size-4)
- Persona pill: `Badge variant="outline"` with `text-xs`, showing persona name (e.g., "Reygadas")
- Role pill: `Badge variant="outline"` with `text-xs`, role-specific background tint:
  - Director: `bg-primary/5 text-primary`
  - Line Producer: `bg-primary/5 text-primary`
  - All roles use neutral primary tint -- no role-specific colors beyond the label text
- Criterion, Weakness, Suggestion: labeled with `text-xs font-medium` for the label, `text-xs text-muted-foreground` for the content
- Checked findings: text gets `line-through text-muted-foreground/60` treatment
- Gap between checkbox and content: 8px (sm)
- Gap between label rows: 4px (xs)
- Bottom border: `border-b border-border` between findings

## Accessibility Notes

- Checkbox findings: each `Checkbox` must have an associated `label` element pointing to the finding text for screen reader association.
- Progress updates during streaming: use `aria-live="polite"` on the progress text container so screen readers announce persona completion.
- Tab addition: the new "Revision" tab follows the existing Tabs pattern which already has proper ARIA roles from shadcn.
- Color: all status colors have text labels alongside color indicators (no color-only communication). Readiness uses both colored dot AND text label.
