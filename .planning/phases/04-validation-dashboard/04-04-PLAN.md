    **Step 1: Create `src/hooks/useValidation.ts`.**

    This hook:
    1. Composes existing data hooks to gather all project data into a ProjectDataSnapshot
    2. Runs `runValidation(snapshot)` from the engine
    3. Runs `computeViabilityScore(snapshot)` for deterministic viability scores
    4. Debounces re-runs by 300ms to prevent flickering during typing (per RESEARCH.md Pitfall 6)
    5. Returns `{ report: ValidationReport | null, viabilityScore: ScoreCategory[], loading: boolean }`

    Data composition:
    - Project metadata: from `useAutoSave(projectId, 'metadata')` OR direct Firestore onSnapshot on `projects/{projectId}/metadata`
    - Team members: Firestore onSnapshot on `projects/{projectId}/team` subcollection
    - Uploaded documents: Firestore onSnapshot on `projects/{projectId}/documents` subcollection
    - Generated documents: from `useGeneratedDocs(projectId)`
    - ERPI settings: Firestore onSnapshot on `erpiSettings/default`
    - Financial data: extracted from project metadata (erpi cash/inkind, third party, eficine amount)
    - Budget/cashflow/esquema totals: extracted from generated docs content (projects/{projectId}/generated/A9a, A9d, E1)

    Use `useMemo` to avoid re-running validation on every render. Debounce with a 300ms delay after the last data change.

    The hook should handle loading states from all data sources -- only run validation when all required data has loaded.

    ```typescript
    export function useValidation(projectId: string): {
      report: ValidationReport | null
      viabilityScore: ScoreCategory[]
      improvements: ImprovementSuggestion[]
      loading: boolean
    }
    ```
