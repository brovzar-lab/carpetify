### 2. COLLAB-04 Partial â€” userProjects Firestore Rule Missing

The `userProjects/{userId}/projects/{projectId}` collection used by Phase 13 for `lastViewedActivity` has no Firestore security rule. In production, the badge count write is silently rejected, so the activity badge never resets to zero after viewing.

**Fix:** Add to `firestore.rules`:
```
match /userProjects/{userId}/{document=**} {
  allow read, write: if request.auth != null && request.auth.uid == userId;
}
```
