## Critical Bugs (Integration-Level)

### BUG-01: useValidation reads Firestore metadata at wrong path
- **File:** `src/hooks/useValidation.ts`
- **Issue:** Reads `projectData.titulo_proyecto` (root) but data is stored at `projectData.metadata.titulo_proyecto`. Also missing subscription to `projects/{id}/financials/data` for EFICINE financial fields.
- **Impact:** 8 validation rules receive zero/empty data. `canExport` falsely returns true. Traffic light shows green for empty projects.
- **Requirements affected:** VALD-01, VALD-02, VALD-04, VALD-05, VALD-07, VALD-14, VALD-16
- **Fix:** Navigate into `metadata.*` for project fields; add Firestore subscription to `financials/data` subcollection.

### BUG-02: ERPI Firestore collection path mismatch
- **File:** `src/hooks/useValidation.ts` line 184
- **Issue:** Reads `erpiSettings/default` (camelCase) but `src/services/erpi.ts` writes to `erpi_settings/default` (snake_case).
- **Impact:** ERPI data always null. VALD-08 cannot validate prior projects.
- **Requirements affected:** VALD-08
- **Fix:** Change to `doc(db, 'erpi_settings', 'default')`.
